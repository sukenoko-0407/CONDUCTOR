CONDUCTOR v2

Skill Overview Report

CONDUCTORは、化合物CSVを出発点として、記述子化、グループ化、解析までをつなぐClaude Code用Skill群です。 このレポートは各機能が何をするものかを短く説明し、JAKサンプルデータでの出力例を示します。

全体像

1

Desc

SMILESからRDKit記述子やfingerprintを作成します。

Output: descriptions/<dataset>/

2

Grouping

構造、fragment、MCS、descriptor clusteringから化合物Groupを作成します。

Output: groups/<dataset>/

3

Analysis Phase1

各descriptor表現空間がpropertyをどれだけ滑らかに表すかを評価します。

Output: analysis/<dataset>/structure_activity_landscape/

4

Analysis Phase2

Groupごとに、活性傾向、構造多様性、解釈価値を評価します。

Output: analysis/<dataset>/group_insight/

3つの主要機能

1. Desc Skill

化合物をベクトル表現に変換します。

  • RDKit 2D descriptors
  • ECFP/FCFP fingerprints
  • MACCS, AtomPair, TopologicalTorsion
  • RDKit fragment counts

2. Grouping Skill

化合物を複数の観点でGroup化します。

  • Murcko, MCS, BRICS/RECAP
  • Butina, hierarchical, DBSCAN
  • Louvain, Leiden, connected components
  • 最終成果物は0/1 membership matrix

3. Analysis Skills

表現とGroupの有用性を評価します。

  • Phase1: Structure-Activity Landscape
  • Phase2: Group Insight Analysis
  • 活性一貫性、濃縮、構造多様性を評価
  • 人が見るための図とreportを生成

JAKサンプルでの実行例

231 input compounds
11 descriptor CSVs
2601 accepted groups
2553 analyzed insight groups

JAKサンプルでは、DescがL01-L11のdescriptor CSVを作成し、Groupingが2601個のGroupを作成しました。 Analysis Phase1では表現空間の比較を行い、Phase2ではGroupごとの活性傾向と構造多様性を評価しました。

Phase2: Group Insight

Phase2は、単に「活性が高いGroup」を探すだけではありません。 「活性が高く、Group内で一貫しており、必要に応じて構造的には多様」という、解釈価値の高い候補を優先します。

使い分け

表現を増やしたい

Descを拡張し、descriptions/に新しいCSV表現を追加します。

Groupを増やしたい

Grouping configで手法やパラメータを調整し、groups/を再生成します。

解釈候補を探したい

Analysis Phase1/2で、表現空間とGroup Insightを確認します。
