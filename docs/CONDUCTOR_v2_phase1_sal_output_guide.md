# CONDUCTOR_v2 Phase1 Output Guide

This document briefly explains the files generated by `cs-conductor-analysis-sal`.

Default output directory:

```text
analysis/<input_csv_stem>/structure_activity_landscape/
```

## Recommended Reading Order

1. `figures/primary_representation_comparison.png`
2. `sal_representation_summary.csv`
3. `figures/local_variance_vs_local_property.png`
4. `figures/sali_distribution_by_representation.png`
5. `sal_warnings.json`

The first comparison should use `primary_comparison_rank`, not raw `median_sali`.

## CSV and JSON Outputs

### `sal_representation_summary.csv`

One row per descriptor representation. This is the main summary table.

Important columns:

- `primary_comparison_rank`: overall representation rank. Smaller is better.
- `primary_comparison_rank_score`: mean rank across the primary kNN property consistency metrics. Smaller is better.
- `median_abs_delta_property_among_knn`: median property difference between each compound and its nearest neighbors. Smaller is better.
- `median_local_property_variance`: median local property variance around each compound. Smaller is better.
- `neighbor_property_autocorrelation`: correlation between each compound's property and its neighbors' properties. Larger is better.
- `median_normalized_sali`: distance-percentile-normalized SALI. Secondary diagnostic only.
- `median_sali`: raw SALI. Use mainly within the same representation, not as the first cross-representation comparison.

### `sal_knn_edges.csv`

One row per directed kNN edge.

Use this when inspecting individual compound-neighbor relationships.

Important columns:

- `compound_id`: source compound.
- `neighbor_compound_id`: nearest-neighbor compound.
- `neighbor_rank`: neighbor rank for the source compound.
- `distance`: raw representation-specific distance.
- `distance_percentile_within_representation`: percentile rank of the edge distance within the same representation.
- `abs_delta_property`: absolute property difference between the two compounds.
- `sali`: raw SALI.
- `normalized_sali`: distance-percentile-normalized SALI.

### `sal_local_metrics.csv`

One row per compound per representation.

Use this to identify compounds whose local neighborhoods are smooth or rugged.

Important columns:

- `local_mean_property`: mean property of the local neighborhood.
- `local_median_property`: median property of the local neighborhood.
- `local_property_variance`: variance of property values in the local neighborhood.
- `median_abs_delta_property_among_knn`: median property gap from the compound to its k nearest neighbors.
- `max_abs_delta_property_among_knn`: largest property gap to any nearest neighbor.

### `sal_sali_distribution.csv`

One row per representation with distribution statistics for raw and normalized SALI.

Use this for SALI diagnostics. Do not use raw SALI alone as the first representation ranking criterion.

Important columns:

- `median`, `p90`, `p95`, `p99`: raw SALI distribution statistics.
- `normalized_median`, `normalized_p90`, `normalized_p95`, `normalized_p99`: normalized SALI distribution statistics.

### `sal_metric_ranking.csv`

Long-format ranking table for each metric.

Use this when checking why a representation ranked high or low.

Important columns:

- `metric_name`: metric being ranked.
- `direction`: whether lower or higher values are better.
- `representation_id`: descriptor representation.
- `metric_value`: actual value.
- `rank`: rank within the metric.

### `sal_manifest.json`

Run metadata.

Use this to confirm what was analyzed.

Includes:

- input CSV path
- descriptor directory
- output directory
- ID column and property column
- kNN `k`
- processed and skipped descriptor files
- primary comparison basis
- output file list

### `sal_warnings.json`

Warnings from the run.

Check this file before interpreting results. Empty warnings mean no detected run-level issues.

Examples of possible warnings:

- descriptor file skipped
- descriptor IDs did not fully match input IDs
- non-numeric property rows excluded
- figure generation failure

## Figure Outputs

### `figures/primary_representation_comparison.png`

Main figure for comparing descriptor representations.

It ranks representations by distance-scale-independent kNN property consistency:

- median absolute property delta among kNN
- median local property variance
- neighbor property autocorrelation

Smaller score is better.

### `figures/local_variance_vs_local_property.png`

Scatter plot of local property ruggedness across the property range.

Axes:

- x: local median property
- y: local property variance

Use this to see whether ruggedness appears mainly in high-, middle-, or low-property regions.

### `figures/auxiliary_metric_summary.png`

Compact summary of auxiliary metrics.

Use this to compare the primary and supporting metrics side by side.

### `figures/sali_distribution_by_representation.png`

Distribution of normalized SALI by representation.

Use this as a cliff diagnostic. A high upper tail suggests that some close-neighbor relationships have large property jumps.

### `figures/sali_ranking.png`

Ranking by median and p90 normalized SALI.

Use this as a secondary diagnostic, not as the first representation comparison.

### `figures/distance_vs_abs_delta_property.png`

Scatter plot of neighbor distance vs absolute property difference.

Use this to inspect whether larger distances tend to correspond to larger property differences. A smoother representation should avoid many high property jumps at very short distances.

## Interpretation Notes

Raw distances are not directly comparable across representation metrics. For example, Tanimoto, cosine, and standardized Euclidean distances have different meanings and distributions.

Therefore:

- use `primary_comparison_rank` for first-pass representation comparison
- use raw `sali` mainly within the same representation
- use `normalized_sali` as a secondary cliff diagnostic
- always check `sal_warnings.json` before drawing conclusions
