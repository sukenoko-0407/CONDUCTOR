# CONDUCTOR_v2 Phase1 Overview: Structure-Activity Landscape

## 1. Purpose

CONDUCTOR_v2 Phase1 adds a new Analysis Skill:

```text
cs-conductor-analysis-sal
```

The analysis name is:

```text
Structure-Activity Landscape
```

Phase1 evaluates whether each descriptor representation generated by the Desc Skill forms a smooth property landscape. It does not use groups. It uses the whole dataset and the descriptor CSV files under `descriptions/<input_csv_stem>/`.

The practical question is:

```text
In this representation space, do nearby compounds tend to have similar property values?
```

If nearby compounds often have large property differences, the representation has a rugged landscape. If nearby compounds have similar property values, the representation has a smoother landscape and is more suitable for that property.

## 2. Position in CONDUCTOR

v1 implemented:

- Desc: descriptor and fingerprint generation
- Grouping: group generation and membership matrix

v2 Phase1 adds:

- Analysis: Structure-Activity Landscape evaluation

Workflow:

```text
input CSV
  -> Desc Skill
       -> descriptions/<input_csv_stem>/*.csv

input CSV + descriptions/<input_csv_stem>/*.csv
  -> cs-conductor-analysis-sal
       -> analysis/<input_csv_stem>/structure_activity_landscape/*
```

Phase1 does not read:

```text
groups/<input_csv_stem>/
```

Group-level analysis is reserved for a later Phase2 Skill.

## 3. Inputs

Phase1 inputs are:

- original input CSV
- descriptor CSV files in `descriptions/<input_csv_stem>/`
- user-facing SAL config

The original input CSV must contain:

- a compound ID column
- a numeric property column, such as `pIC50`

Descriptor CSVs must contain:

- a compound ID column
- numeric descriptor or fingerprint columns

Known non-feature columns such as `canonical_smiles`, `mol_parse_ok`, and `descriptor_error` are ignored.

## 4. User-Facing Config

The metric definitions and analysis parameters are controlled by:

```text
config/CONDUCTOR_v2_sal_phase1_config.json
```

This config defines:

- property column detection
- kNN parameter `k`
- SALI epsilon
- descriptor file matching rules
- metric per descriptor representation
- scaling per descriptor representation
- figure output settings

This keeps the Analysis Skill loosely coupled to Desc. When new descriptor files are added, users can add a new representation block to the config without editing analysis code.

## 5. Core Metrics

Phase1 uses continuous landscape metrics. It intentionally avoids binary Cliff counting because cliff thresholds are target-dependent and require manual tuning.

The main purpose is to compare descriptor representation methods. Because raw distances have different meanings and scales across metrics such as Tanimoto, cosine, and standardized Euclidean, raw SALI is not used as the first cross-representation ranking criterion.

Primary representation-comparison metrics are distance-scale-independent kNN property consistency metrics:

1. median absolute property delta among kNN
2. local property variance
3. neighbor property autocorrelation

These metrics ask:

```text
When each representation chooses nearest neighbors by its own appropriate metric,
how similar are the property values of those neighbors?
```

SALI is retained as a cliff diagnostic.

In CONDUCTOR_v2 Phase1, raw SALI means generalized SALI:

```text
SALI(i,j) = |property_i - property_j| / max(distance(i,j), epsilon)
```

It is computed on kNN edges in each descriptor representation space.

Lower raw SALI means a smoother local property landscape within the same representation. However, raw SALI should not be treated as a direct first-rank comparison value across representations because the distance denominator is not commensurate across representation metrics.

For secondary cross-representation diagnostics, Phase1 also reports distance-percentile-normalized SALI:

```text
normalized_sali(i,j) =
  |property_i - property_j| / max(distance_percentile_within_representation(i,j), epsilon)
```

## 6. kNN Policy

For each descriptor representation:

1. Extract numeric feature columns.
2. Apply representation-specific scaling.
3. Compute distances using the metric defined in config.
4. For each compound, find `k` nearest neighbors.
5. Compute property consistency metrics from kNN edges.
6. Compute raw SALI and normalized SALI as cliff diagnostics.

Default:

```json
{
  "knn": {
    "k": 10
  }
}
```

If the dataset has fewer compounds, the effective value is:

```text
effective_k = min(k, n_valid_compounds - 1)
```

## 8. Visualization Policy

Phase1 is designed to be evaluated primarily through figures.

Required primary comparison visualization:

- primary representation comparison by distance-independent kNN property metrics

Required SALI diagnostic visualizations:

- SALI distribution by representation
- SALI ranking by representation
- distance vs absolute property delta scatter or density plot

Required auxiliary visualizations:

- local property variance vs local mean or median property scatter
- one compact summary plot for auxiliary metrics

Auxiliary metrics should generally have one strong visualization each. At most two figures per auxiliary metric should be produced by default.

Top/Middle/Low property binning is optional, not required. The preferred visualization for activity-regime effects is:

```text
x-axis: local mean or median property
y-axis: local property variance
point: compound
facet/color: representation
```

This preserves continuous property information while showing whether the landscape becomes rugged in high- or low-property regions.

## 10. Interpretation Direction

Primary comparison lower is better:

- median absolute property delta among kNN
- median local property variance

Primary comparison higher is better:

- neighbor property autocorrelation

SALI interpretation:

- raw SALI is mainly a within-representation cliff diagnostic
- normalized SALI is a secondary cross-representation diagnostic
- raw median/p90/p95 SALI should not override the primary comparison rank

Other supporting metric:

- distance-property Spearman correlation

Spearman correlation should be interpreted carefully. A high positive value means that larger representation distances tend to correspond to larger property differences, which is desirable for landscape consistency.
