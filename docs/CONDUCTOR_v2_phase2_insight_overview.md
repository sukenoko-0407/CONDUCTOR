# CONDUCTOR_v2 Phase2 Overview: Group Insight Analysis

## 1. Purpose

CONDUCTOR_v2 Phase2 adds a new Analysis Skill:

```text
cs-conductor-analysis-insight
```

The analysis name is:

```text
Group Insight Analysis
```

Phase2 uses the groups generated by the Grouping Skill to find interpretable group-level structure-activity insights. Phase1 evaluated descriptor representations across the whole dataset. Phase2 narrows the scope to individual groups and asks whether any group shows a sharp, useful, and interpretable property trend.

The practical question is:

```text
Which groups contain compounds with a clear property trend, and why might that group be informative?
```

## 2. Position in CONDUCTOR

v1 implemented:

- Desc: descriptor and fingerprint generation
- Grouping: group generation and membership matrix

v2 Phase1 implemented:

- Analysis: Structure-Activity Landscape evaluation across descriptor representations

v2 Phase2 adds:

- Analysis: Group-level insight detection using Grouping outputs

Workflow:

```text
input CSV
  -> Desc Skill
       -> descriptions/<input_csv_stem>/*.csv

input CSV
  -> Grouping Skill
       -> groups/<input_csv_stem>/group_membership_matrix.csv
       -> groups/<input_csv_stem>/group_registry.json
       -> groups/<input_csv_stem>/group_relations.json

input CSV + groups/<input_csv_stem>/* + optional descriptions/<input_csv_stem>/L02_ecfp4_bit.csv
  -> cs-conductor-analysis-insight
       -> analysis/<input_csv_stem>/group_insight/*
```

Phase2 reads Grouping outputs. It does not create new groups.

## 3. Core Idea

An informative group is not just a group with high activity. Phase2 should prioritize groups that are:

- activity-enriched relative to the full dataset
- internally consistent in property values
- large enough to be credible
- structurally interpretable or traceable to a grouping method
- not simply redundant with a higher-ranked group
- optionally, structurally diverse by ECFP4 Tanimoto similarity while still showing high activity

The last point is important. A group where compounds are structurally very similar and active is useful but often unsurprising. A group where compounds have low mean ECFP4 Tanimoto similarity but still share high activity may reveal a more valuable insight, such as scaffold hopping or a non-obvious shared activity pattern.

## 4. Inputs

Required inputs:

- original input CSV
- `groups/<input_csv_stem>/group_membership_matrix.csv`
- `groups/<input_csv_stem>/group_registry.json`

Recommended optional inputs:

- `groups/<input_csv_stem>/group_relations.json`
- `groups/<input_csv_stem>/group_membership.csv`
- `descriptions/<input_csv_stem>/L02_ecfp4_bit.csv`
- `analysis/<input_csv_stem>/structure_activity_landscape/sal_representation_summary.csv`

The original input CSV must contain:

- compound ID column
- numeric property column, such as `pIC50`

By default, Phase2 assumes higher property values are better, which is appropriate for pIC50-like values. This must be configurable because raw IC50-like values have the opposite direction.

## 5. Main Analysis Axes

### 5.1 Activity Enrichment

Detect groups whose property distribution is shifted relative to the global dataset.

Examples:

- high-activity enriched group
- low-activity enriched group
- group median property above global median
- group mean property above global mean
- high-active fraction above global high-active fraction

High and low activity should be defined by config. Default:

```text
high activity = top 20% of property values
low activity = bottom 20% of property values
```

When `higher_is_better` is false, the high and low directions are reversed.

### 5.2 Activity Consistency

Detect groups whose member compounds have consistent property values.

Important metrics:

- property variance
- property standard deviation
- median absolute deviation
- interquartile range
- median pairwise absolute property delta

A group with high median activity but high variance may be less interpretable than a group with slightly lower median activity but much better consistency.

### 5.3 Structural Diversity by ECFP4 Tanimoto

Optionally compute within-group pairwise ECFP4 bit Tanimoto similarity using:

```text
descriptions/<input_csv_stem>/L02_ecfp4_bit.csv
```

Important metrics:

- mean ECFP4 Tanimoto similarity
- median ECFP4 Tanimoto similarity
- p25 and p75 Tanimoto similarity
- structural diversity score = `1 - mean_ecfp4_tanimoto`

Low mean ECFP4 Tanimoto means the group is structurally diverse by this fingerprint.

The key Phase2 insight category is:

```text
high activity + activity consistency + low mean ECFP4 Tanimoto
```

This should produce a separate ranking:

```text
structurally_diverse_active_group_rank
```

### 5.4 Interpretability

Interpretability depends on group provenance.

More directly interpretable group sources:

- MCS groups
- Murcko scaffold groups
- BRICS fragment groups
- RECAP fragment groups
- human/user-defined groups

Less direct but still useful group sources:

- descriptor clustering groups
- similarity clustering groups
- meta groups

The output should preserve `group_id`, `group_label`, `group_type`, `group_source`, and key definition fields from `group_registry.json`.

### 5.5 Redundancy and Overlap

Grouping can produce many overlapping groups. Phase2 should identify when a group is mostly redundant with a higher-ranked group.

Important metrics:

- maximum Jaccard overlap with any higher-ranked group
- overlapping group count above a configurable threshold
- redundant flag

Redundancy should lower priority but should not delete the group from output.

## 6. Recommended Ranking Outputs

Phase2 should produce multiple ranks instead of a single black-box score:

```text
activity_enriched_group_rank
structurally_diverse_active_group_rank
consistent_group_rank
interpretable_group_rank
insight_priority_rank
```

Recommended interpretation:

- `activity_enriched_group_rank`: strongest high-activity or low-activity enrichment
- `structurally_diverse_active_group_rank`: high-activity, consistent, structurally diverse groups
- `consistent_group_rank`: groups with low property dispersion
- `interpretable_group_rank`: groups with clear structure or source interpretation
- `insight_priority_rank`: pragmatic overall priority, with redundancy penalty

The individual metric columns should remain visible so the user can understand why a group ranked highly.


## 8. Visualization Policy

Required figures:

- activity enrichment vs group size
- group median property distribution for top groups
- high-activity enrichment vs mean ECFP4 Tanimoto
- structural diversity vs activity consistency
- group overlap heatmap for top groups
- group source/type summary

The most important Phase2 figure should show the surprising active-diverse quadrant:

```text
x-axis: mean ECFP4 Tanimoto within group
y-axis: high-activity enrichment or group median property
point size: group size
color: group source or group type
```

The upper-left region is especially interesting:

```text
low ECFP4 similarity + high activity enrichment
```

## 9. Interpretation Direction

High-value groups generally have:

- high activity enrichment
- low property variance
- sufficient group size
- low redundancy with higher-ranked groups
- clear group provenance
- optionally low mean ECFP4 Tanimoto despite high activity

Low-priority groups generally have:

- property distribution similar to global background
- very high within-group variance
- very small group size
- strong overlap with many higher-ranked groups
- no clear structural or grouping interpretation

Phase2 should not make mechanistic SAR claims. It should produce ranked evidence and concise interpretation prompts for the user.

## 10. Relationship to Phase1

Phase1 answers:

```text
Which descriptor representations create a smooth property landscape over the whole dataset?
```

Phase2 answers:

```text
Which specific groups show useful, interpretable property trends?
```

Phase2 may read Phase1 outputs for context, but Phase2 must remain usable without Phase1 results as long as Grouping outputs and a property column are available.
